# 个人介绍网页设计文档

## 项目目标

基于 `D:\XKKProjectLIst\webpage\5_2_徐珂珂简历2024.0305.pdf` 中的内容，制作一个单页长滚动的个人介绍网页。网页定位为学术展示型个人主页，面向老师、评审或学术相关场景，整体风格简约、克制，带有少量细腻动画，重点突出竞赛获奖与综合能力，同时完整保留核心简历信息。

## 已确认的设计方向

- 使用场景：老师或学术场景浏览
- 页面结构：单页长滚动
- 形象素材：不使用真人照片，优先整体高级感
- 首屏优先级：先展示竞赛获奖与综合能力
- 视觉气质：克制学术感
- 信息密度：尽量完整保留原始内容
- 推荐落地方向：学术主页型，而不是纯简历型或纯作品集型

## 页面整体结构

页面按自上而下的阅读逻辑组织为六个连续模块：

1. 首屏 Hero
2. 个人简介
3. 完整奖项与荣誉
4. 科研项目与学术实践
5. 论文、专利与专业技能
6. 任职经历与志愿服务

这种排序不完全照搬简历原顺序，而是优先建立“成果丰富、综合能力强”的第一印象，再逐步展开科研、产出和服务经历，符合网页浏览习惯。

## 模块设计

### 1. 首屏 Hero

首屏承担身份建立和快速概览的功能。

- 左侧展示姓名、学校专业、简短个人定位和一句网页化的个人介绍
- 右侧展示抽象几何图形与数据摘要卡片
- 数据摘要固定包含四项：
  - 15+ 竞赛奖项
  - 3 篇论文成果
  - 2 项专利申请
  - 多项科研项目参与或主持

首屏不使用真人照片，改用抽象几何图形、柔和色块和层叠卡片来营造高级感和学术气质。

### 2. 个人简介

该模块用于快速呈现人物特征和综合印象。

- 使用 4 到 5 行精炼文本概括性格与能力特点
- 表述重点包括：
  - 学习和工作认真细致
  - 乐于助人，具备良好协作意识
  - 写作整理能力较强
  - 愿意接受新挑战，具有社会责任感
- 搭配 3 个关键词标签，内容定为“科研实践”“组织协作”“写作整理”

该区域需要比简历原文更像网页语言，避免直接堆叠原句。

### 3. 完整奖项与荣誉

这是全页最重要、信息量最大的模块，核心目标是在完整保留内容的前提下，降低阅读压迫感。

展示方式：

- 第一层：概览卡片
  - 国家级奖项
  - 省级奖项
  - 校级奖项
  - 公开报道与荣誉
- 第二层：完整奖项列表
  - 保留主要获奖原文
  - 采用分组和卡片化排版
  - 列表按重要度和阅读节奏整理，不直接形成大段文字墙

需纳入的代表性内容包括但不限于：

- 第七届全国大学生基础医学创新研究暨实验设计论坛国家银奖
- 第七届全国大学生基础医学创新研究暨实验设计论坛中南部赛区一等奖
- 第四届全国大学生智能技术应用大赛国家一等奖
- 第二届全国大学生智能技术应用大赛国家二等奖
- 第八届互联网+大学生创新创业大赛河南省二等奖
- 第三届全国大学生智能技术应用大赛国家二等奖、三等奖
- 第十五届挑战杯河南省大学生课外学术科技作品竞赛省级三等奖
- 第二届康复产业创新创业大赛河南赛区三等奖、明日之星
- 第六届互联网+大学生创新创业大赛河南省三等奖
- 第十五届挑战杯校级一等奖
- 第八届互联网+校级一等奖
- 第七届互联网+校级三等奖
- 优秀学生朗读者校级三等奖
- 省级、校级、院级平台公开报道
- 综合奖学金、优秀学生干部、优秀应届毕业生、优秀实习生、最美志愿者等荣誉

页面实现时需要在“完整展示”和“视觉整洁”之间取得平衡，优先用结构优化解决拥挤问题，而不是删减信息。

### 4. 科研项目与学术实践

这一模块用于体现持续研究能力与项目参与深度，采用纵向时间线展示。

展示重点包括：

- 主持：混合现实在儿童自闭症患者注意力康复训练的应用
- 主持：焦虑症患者心理治疗的增强现实场景构建
- 参与：基于脑机接口的脑卒中患者行走康复训练系统的设计
- 参与：河南省民办教育协会相关课题
- 其他科研立项与实践经历

每条项目统一展示：

- 项目身份：主持或参与
- 项目名称
- 项目编号（原简历中有编号的项目必须展示编号）
- 研究方向关键词

这样可以让页面更接近正式学术主页，而不是简单的经历堆叠。

### 5. 论文、专利与专业技能

该模块采用左右双栏排版，形成“学术产出 + 能力基础”的组合展示。

  - 一种辅助盲人出行的避障导航眼镜

右栏内容：

这样可以在一个模块里同时说明“已有成果”和“能力来源”。

### 6. 任职经历与志愿服务

该模块作为后段补充，用于强化组织能力、责任心和综合素养，不抢前面成果模块的主视觉重心。

应包含：

- 小班班长
- 朋辈导师
- 医工研创工作室外联部部长
- 第七次人口普查普查员

内容表达上突出：

- 组织协调能力
- 团队辅导与带队能力
- 服务意识与社会责任感
- 与老师、同学协作中的执行力

## 视觉设计

### 配色

整体采用偏灰绿的学术色系，避免强烈科技蓝或过暖的作品集配色。

- 背景色：米白、浅灰绿
- 主色：深墨绿色
- 辅助色：浅灰、雾感绿色

该配色用于建立安静、可信、克制的专业感，也更贴合生物医学工程背景。

### 版式

- 桌面端使用双栏与单栏穿插的节奏布局
- 模块之间保留较大留白，增强层次
- 标题具有识别度，正文保持高可读性
- 页面顶部提供简洁导航，支持跳转到各模块

### 图形元素

- 不使用真人照片
- 使用抽象几何图形、渐变色块、线性背景装饰
- 图形主要服务于氛围建立，不应干扰文字阅读

## 动画与交互

整体动画原则为“少量、克制、服务阅读”，避免炫技。

计划使用以下三类动画：

1. 首屏加载动画
   - 背景几何图形缓慢浮动
   - 摘要卡片轻微淡入
2. 滚动进入动画
   - 各模块在进入视口时轻微上移并淡入
3. 悬停反馈
   - 奖项卡片、时间线节点和导航链接在鼠标悬停时产生轻微高亮或阴影变化

交互功能包括：

- 顶部导航平滑滚动
- 长奖项区域支持更清晰的分组浏览
- 移动端下模块自动切换为单栏布局

## 技术落地方案

页面使用轻量静态站方式实现，不引入重型框架。

文件结构如下：

- `index.html`：页面结构与内容骨架
- `styles.css`：配色、排版、动画、响应式布局
- `script.js`：平滑滚动、滚动进入动画与少量交互逻辑
- `assets/`：装饰图形资源统一放置

选择原生 `HTML + CSS + JavaScript` 的原因：

- 当前工作区为空白起点，页面目标明确
- 页面以展示为主，不需要复杂数据状态管理
- 便于后续快速预览、修改和交付

## 内容处理原则

页面内容分两种处理方式：

1. 首屏与个人简介
   - 转写为更自然、更适合网页阅读的表达
2. 奖项、项目、论文、经历等事实信息
   - 尽量保留原始内容
   - 主要通过分组、结构和版式优化可读性

网页不应成为 PDF 的机械复制版，而应成为结构更清晰、浏览更友好的展示页面。

## 验收标准

页面完成后应满足以下标准：

- 第一屏能够快速传达身份、方向和优势
- 奖项信息完整，但浏览体验不压迫
- 科研项目、论文、技能与经历层次清晰
- 整体风格简约、克制，带有少量耐看的动画
- 桌面端和移动端均能正常展示

## 非目标

本次页面不包含以下内容：

- 多页面站点结构
- 后端服务或数据库
- 表单提交功能
- 复杂筛选系统或管理后台
- 过强的科技炫光、重视觉特效或照片墙式展示
